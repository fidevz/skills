# Mission Control

Autonomous project orchestration system. Routes tasks through a 5-stage pipeline (backlog → research_planning → in_progress → validating → done), delegates work to team agents via Claude sub-agents, and runs on cron with push notifications.

## When invoked

1. Check if `mission-control-tasks/` exists in the current working directory.
   - **If NOT**: the project hasn't been set up yet. Run:
     ```bash
     bash "$(dirname "$0")/../skills/mission-control/setup.sh"
     ```
     Or, if the skill lives at a known path, instruct the user to run `setup.sh` from the skill directory.
   - **If YES**: proceed with the normal Mission Control workflow below.

2. Read `mission-control-tasks/state.json` immediately to understand the current state.

3. Determine what action the user requested and execute it using the commands in `mission-control-tasks/mission-control.md` (the project-specific command reference generated by setup).

4. Confirm the result by showing the updated state.

## Quick reference (before setup.sh has been run)

If `mission-control-tasks/mission-control.md` doesn't exist yet (first-time setup):

### Activate cron
```bash
(crontab -l 2>/dev/null; echo "*/5 * * * * $(pwd)/mission-control-tasks/orchestrate.sh > /dev/null 2>&1") | crontab -
```

### Open dashboard
```bash
cd "$(pwd)" && python3 -m http.server 4000 &
echo "http://localhost:4000/mission-control-tasks/dashboard.html"
```

### Read state
```bash
jq '.objective | {id, title, status}' mission-control-tasks/state.json
jq '.agents' mission-control-tasks/state.json
```
